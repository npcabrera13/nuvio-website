# Nuvio — Technical Architecture

> Complete technical documentation of the Nuvio streaming service.

---

## 🏗️ System Overview

Nuvio is a three-tier system with a shared Firestore database:

```
┌─────────────────────────────────────────────────────────────────┐
│                        Firestore (Firebase)                     │
│                                                                 │
│  Single collection: customers/{tokenId}                         │
│  Fields: nuvioEmail, nuvioPassword, name, status, expiresAt,   │
│          createdAt, assignedTo, notes                           │
└────────────────────────┬────────────────────────────────────────┘
                         │
        ┌────────────────┼────────────────┐
        ▼                ▼                ▼
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│   Website    │ │  Streaming   │ │ Admin Panel  │
│  Cloudflare  │ │  API (Vercel)│ │  (Vercel)    │
│  Pages       │ │              │ │              │
│              │ │  Stremio     │ │  Token CRUD  │
│  Next.js 16  │ │  proxy +     │ │  Stats       │
│  + Functions │ │  gatekeeper  │ │  Bulk create │
└──────────────┘ └──────────────┘ └──────────────┘
```

## 🗄️ Database Schema

### Collection: `customers/{tokenId}`

**Document ID** = the token string (e.g. `nuvio_abc123x`), generated by the admin panel as `nuvio_` + 7 random base36 chars.

| Field | Type | Set by | Purpose |
|-------|------|--------|---------|
| `nuvioEmail` | string | Admin | Real Nuvio.tv account email |
| `nuvioPassword` | string | Admin | Real Nuvio.tv account password (plaintext) |
| `name` | string | Website (on verify) / Admin | Customer's email (empty = available) |
| `status` | string | Admin / Website | `available` \| `active` \| `blocked` \| `expired` |
| `expiresAt` | Timestamp | Website (on verify) / Admin | Subscription expiry |
| `createdAt` | serverTimestamp | Admin | When the token was created |
| `assignedTo` | string \| null | Admin | Firebase Auth UID of the assigned user |
| `notes` | string | Admin | Free-text notes |

### Status lifecycle

```
available → active → expired
              ↑         │
              └─────────┘ (renew via PayMongo)

admin can block at any time:  * → blocked
admin can unassign:           active/expired → available (clears name + assignedTo)
```

**Key rule:** The website ONLY assigns tokens where `status == "available"` AND `name` is empty. This prevents double-assignment.

### Auth Flow (auth-context.tsx)

```typescript
// signup: creates Firebase Auth user + sends verification email (0 Firestore)
async function signup(email, password) {
  const cred = await createUserWithEmailAndPassword(auth, email, password);
  if (isAdmin(email)) {
    await assignTokenToUser(cred.user, cred.user.email);  // skip verification
    return { needsVerification: false };
  }
  await fetch("/api/send-verification", { ... });  // sends JWT email
  return { needsVerification: true };
}

// completeVerification: called from /verify page AFTER email click
// This is where the Nuvio account actually gets assigned
async function completeVerification() {
  await assignTokenToUser(user, user.email);
  await fetchProfile(user);
}

// assignTokenToUser: queries available token + assigns it
async function assignTokenToUser(firebaseUser, userEmail) {
  const q = query(collection(db, "customers"),
    where("status", "==", "available"), limit(1));
  const snapshot = await getDocs(q);
  if (snapshot.empty) throw new Error(ACCOUNTS_FULL_ERROR);

  const tokenDoc = snapshot.docs[0];
  if (tokenDoc.data().name?.trim()) throw new Error(ACCOUNTS_FULL_ERROR); // safety

  const expires = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000); // 7 days
  await updateDoc(doc(db, "customers", tokenDoc.id), {
    name: userEmail,
    status: "active",
    expiresAt: Timestamp.fromDate(expires),
  });
  await updateProfile(firebaseUser, { displayName: tokenDoc.id }); // 0 Firestore
}
```

### Firestore reads/writes per user action

| Action | Reads | Writes |
|--------|-------|--------|
| Signup | 0 | 0 (just sends email) |
| Verify email (click link) | 1 (query available) | 1 (assign token) |
| Dashboard load | 1 (`customers/{displayName}`) | 0 |
| Login (returning) | 1 (fetchProfile) | 0 |
| Resend verification | 0 | 0 |
| **Total per new user** | **2** | **1** |

### Email Verification (JWT, zero Firestore)

```typescript
// send-verification.ts (Cloudflare Function)
// Signs: { uid, email, exp } with HMAC-SHA256 + VERIFICATION_SECRET
const payload = JSON.stringify({ uid, email, exp: Date.now() + 86400000 });
const signature = await sign(payloadB64, secret);  // Web Crypto API
const token = `${payloadB64}.${signature}`;
// → sends email with /verify?token=...

// verify-email.ts (Cloudflare Function)
// Verifies signature WITHOUT touching Firestore
const valid = await verifySignature(payloadB64, signature, secret);
if (!valid) return { error: "Invalid token" };
if (Date.now() > payload.exp) return { error: "Expired" };
return { success: true, email: payload.email, uid: payload.uid };

// verify/page.tsx (client)
// Calls completeVerification() which assigns the token (1 read + 1 write)
```

### How it works

The streaming API is a **reverse proxy + gatekeeper** for Stremio. It sits between the customer's Stremio app and the upstream addon providers (Cinemeta, Torrentio, etc.).

```
Customer's Stremio app
  → requests: nuviostreamapi.vercel.app/{token}/stream/movie/tt0111161.json
  → Vercel rewrites to: /api/proxy?token={token}&prefix=stream&p=movie/tt0111161.json
  → proxy.js validates the token against Firestore
  → If valid: fetches the stream from Torrentio, rebrands "Torrentio" → "Nuvio Bundle"
  → Returns the stream list to Stremio
  → Customer watches the movie
```

### URL Routing (vercel.json rewrites)

| Pattern | Maps to | Purpose |
|---------|---------|---------|
| `/manifest.json` | `/api/proxy?prefix=manifest` | No-token manifest (empty) |
| `/:token/manifest.json` | `/api/proxy?token=X&prefix=manifest` | Master bundle manifest |
| `/:token/stream/:p` | `/api/proxy?token=X&prefix=stream&p=...` | Stream request (validated) |
| `/:token/catalog/:p` | `/api/proxy?token=X&prefix=catalog&p=...` | Catalog (no validation) |
| `/:token/:addon/stream/:p` | `/api/proxy?token=X&addon=Y&prefix=stream&p=...` | Single-addon passthrough |

### Token Validation (proxy.js)

```javascript
const TOKEN_REGEX = /^[a-zA-Z0-9_-]{4,128}$/;
const CUSTOMER_TTL = 5 * 60_000; // 5-minute in-memory cache

async function getCustomerData(token) {
  // Check cache first (5-min TTL)
  const hit = _customerCache.get(token);
  if (hit && Date.now() - hit.time < CUSTOMER_TTL) return hit.data;

  // Read from Firestore
  const snap = await getDoc(doc(db, "customers", token));
  const data = snap.exists() ? snap.data() : null;
  _customerCache.set(token, { data, time: Date.now() });
  return data;
}

function isBlocked(data) {
  if (!data) return true;                    // token doesn't exist
  if (data.status !== "active") return true; // blocked/disabled
  if (data.expiresAt && Date.now() > data.expiresAt.toMillis()) return true; // expired
  return false;
}
```

**Important:** Only `stream/*` requests validate the token. Manifest, catalog, and meta endpoints are zero-read fast paths (no Firestore).

### Admin Firestore Operations

| Operation | Firestore call |
|-----------|---------------|
| Load all tokens | `getDocs(collection(db, "customers"))` |
| Create single | `setDoc(doc(db, "customers", tokenId), {...})` |
| Bulk create | Loop: `setDoc` per line |
| Edit | `updateDoc(doc(db, "customers", tokenId), {...})` |

## 🔗 How the Three Parts Connect

```
1. ADMIN creates a token in the admin panel
   → admin.js writes to Firestore: customers/{tokenId}
   → Token is "available"

2. CUSTOMER signs up on the website
   → Website creates Firebase Auth user
   → Website sends verification email (JWT signed)

3. CUSTOMER clicks the email link
   → /verify page verifies JWT signature (0 Firestore)
   → /verify calls completeVerification()
   → completeVerification queries Firestore for an available token
   → Updates the token: name = customer email, status = active, expiresAt = +7 days
   → Stores tokenId in user.displayName (Firebase Auth)
   → Redirects to /dashboard

4. CUSTOMER views dashboard
   → Dashboard reads customers/{user.displayName} (1 Firestore read)
   → Shows Nuvio credentials + countdown + channels

5. CUSTOMER adds the Stremio addon
   → Copies the addon URL: nuviostreamapi.vercel.app/{tokenId}/manifest.json
   → Adds it to their Stremio app
   → Stremio fetches the manifest (0 Firestore — hardcoded)
   → Stremio fetches catalogs (0 Firestore — proxied from upstream)

6. CUSTOMER watches a movie
   → Stremio requests: /{token}/stream/movie/tt0111161.json
   → proxy.js validates the token against Firestore (cached 5 min)
   → If valid: fetches stream from Torrentio, rebrands to "Nuvio Bundle"
   → Returns stream list to Stremio
   → Customer watches

7. CUSTOMER's trial expires
   → proxy.js sees expiresAt < now → returns empty streams
   → Stremio shows "no streams available"
   → Customer goes to dashboard → sees "Reactivate" screen
   → Pays via PayMongo → you manually extend in admin panel
```

## ⚠️ Known Limitations & Security Notes

### Security
1. **No Firebase Admin SDK** — both the API and admin panel use the public web SDK. Security depends entirely on Firestore rules.
2. **Admin password** is a single SHA-256 hash in client-side JS. Anyone who views `admin.js` can see the hash and brute-force it offline.
3. **Nuvio passwords stored in plaintext** in Firestore.
4. **5-minute token cache** means a blocked customer keeps streaming access for up to 5 minutes after being blocked.
5. **No rate limiting** on the API or signup endpoint.

### Operational
1. **PayMongo webhook doesn't auto-renew** — payments are logged but accounts must be extended manually.
2. **M3U playlists contain time-limited auth tokens** — PH-local DASH streams break when upstream auth expires.
3. **`puppeteer-core` is an unused dependency** — can be removed to shrink deploy.
4. **Addon list is duplicated** between `api/proxy.js` and `admin.js` — changes require editing both.
5. **No `firestore.rules` in either repo** — rules are managed in Firebase Console.

### Cost
1. **Vercel free tier:** 100GB bandwidth/month, 100GB-hours serverless function execution
2. **Cloudflare Pages free tier:** unlimited bandwidth, 500 builds/month
3. **Firebase free tier:** 50K reads/day, 20K writes/day, 1GB storage
4. **PayMongo fees:** GCash 1.5%, cards 3.5% + ₱15
